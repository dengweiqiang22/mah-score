# 开发任务书（TASK）

> 本文档定义 mah-score v0.14.1 的候选开发任务。
>
> v0.14.1 的目标是在 v0.14.0 高频记录区交互清晰度优化基础上，继续减少重复提示和视觉噪音，让牌桌旁录入更轻、更快。
>
> 如本文档与 AGENTS.md、ARCHITECTURE.md、DECISIONS.md 存在冲突，以后三者为准。

---

# 一、当前版本

**版本：v0.14.1**

主题：

高频记录区信息降噪与状态表达优化。

背景：

v0.14.0 已围绕高频记录区做了基础优化：

- 当前步骤提示更紧凑；
- 玩家卡片保留头像、昵称和录入角色；
- 事件入口保持固定网格；
- 高频录入流程仍由现有状态机驱动。

当前继续观察到：

- 玩家卡片内的禁用原因提示容易造成视觉噪音；
- “当前步骤”提示和玩家区标题绑定较强，容易被理解为只属于玩家区；
- “取消当前录入”文字按钮在移动端占据标题栏空间；
- 录入状态文案来源较多，需要明确职责边界，避免重复提示。

---

# 二、目标

本次目标是让高频记录区的信息层级更清楚：

- 玩家卡片只表达玩家身份和当前录入角色；
- 禁用原因不在卡片内重复解释；
- 当前步骤提示作为全局录入状态，而不是玩家区局部说明；
- 取消当前录入入口更轻量；
- 保持现有录入状态机和业务校验不变。

---

# 三、范围约束

## 不改变

- API；
- Event Sourcing；
- Redis 数据模型；
- domain 计分逻辑；
- 录入状态机；
- 计分规则；
- 同步机制；
- 玩家头像数据模型；
- 当前头像资源映射；
- 事件提交和撤销逻辑。

## 可以改变

- 高频记录区局部布局；
- 当前步骤提示展示位置和样式；
- 玩家卡片内部文案；
- 取消当前录入按钮样式；
- 高频记录区文案职责划分；
- `RoomPage` 内部 UI 渲染结构。

---

# 四、实施顺序

## 第一轮：移除玩家卡片禁用原因提示

目标：

玩家卡片不再展示禁用原因，只保留头像、昵称和当前录入角色。

要求：

- 删除卡片内以下提示：
  - `本局已胡牌`
  - `不能选择同一玩家`
  - `等待完成当前录入`
- 保留卡片禁用态；
- 保留选中态；
- 保留角色标签：
  - `胡牌者`
  - `自摸玩家`
  - `杠牌玩家`
  - `点炮玩家`
  - `引杠玩家`
  - `已选玩家`
- 不改变点击控制和业务校验。

验收：

- 已胡牌玩家仍置灰且不可点击；
- 点炮者不能选择胡牌者本人；
- 选番或流局确认阶段玩家卡片仍不可点击；
- 卡片内不再出现禁用原因提示。

## 第二轮：调整当前步骤提示层级

目标：

让当前步骤提示表达整个高频记录区的下一步动作。

要求：

- 不删除当前步骤提示；
- 不增加新的业务状态；
- 继续复用现有 `quickScoreStepMessage`；
- 避免提示只像“玩家”标题的附属信息；
- 375px 竖屏下不增加明显高度负担。

建议方向：

```text
高频记录
当前：请选择点炮玩家
```

或：

```text
高频记录 · 请选择点炮玩家
```

具体实现以实际移动端效果为准。

## 第三轮：弱化取消当前录入入口

目标：

减少标题栏右侧文字按钮的视觉占用。

要求：

- 保留取消能力；
- 仅在非 idle 状态显示；
- 按钮触控面积足够；
- 不使用复杂动画；
- 不新增依赖。

建议方向：

- 使用小尺寸 `X` 或 `RotateCcw` 图标按钮；
- 保留可访问名称；
- 保持低权重，不抢主流程视线。

## 第四轮：整理录入文案职责

目标：

明确不同文案的职责，避免页面同时出现多处重复说明。

要求：

- `quickScoreStepMessage`：只表达下一步；
- `getQuickScoreSummary`：只表达已选结果；
- `scoreFeedbackMessage`：只表达提交结果；
- `getQuickScoreMissingMessage`：只用于提交缺失信息时的错误提示；
- 不把同一含义同时放进多个视觉区域。

---

# 五、验收标准

功能验收：

- 玩家选择流程可用；
- 自摸录入可用；
- 点炮录入可用；
- 直杠、补杠、暗杠可用；
- 流局确认可用；
- 已胡牌玩家不能继续作为胡牌玩家；
- 点炮玩家不能和胡牌玩家相同。

视觉验收：

- 玩家卡片更干净；
- 玩家卡片不再出现禁用原因小字；
- 当前步骤提示仍清楚；
- 标题栏不拥挤；
- 375px 竖屏下核心操作不显得更复杂。

代码验收：

- 不新增依赖；
- 不使用 `any`；
- 不修改 domain、API、Redis 数据模型；
- `npm run lint` 通过；
- `npm run typecheck` 通过；
- `npm run build` 通过。

---

# 六、不能接受的结果

以下情况视为任务未完成：

- 删除当前步骤提示，导致用户不知道下一步；
- 为 UI 降噪修改业务规则；
- 为 UI 降噪修改录入状态机；
- 已胡牌玩家看起来仍可点击；
- 选中玩家和责任玩家状态不清楚；
- 取消当前录入入口消失；
- 页面视觉变得更拥挤。
