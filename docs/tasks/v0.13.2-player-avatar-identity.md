# 开发任务书（TASK）

> 本文档定义 mah-score v0.13.2 的候选开发任务。
>
> v0.13.2 的目标是在房间页结构和低频功能整理完成后，为玩家加入房间流程增加头像选择，并在房间内用头像辅助识别玩家。
>
> 如本文档与 AGENTS.md、ARCHITECTURE.md、DECISIONS.md 存在冲突，以后三者为准。

---

# 一、当前版本

**版本：v0.13.2**

主题：

玩家头像与身份识别。

当前问题：

- 房间内玩家目前主要通过昵称和首字母识别；
- `RoomScoreStrip` 头像区域只是首字母占位；
- 用户已经准备了 `frontend/src/assets/avatars/` 下的 SVG 卡通头像；
- 需要在进入房间时选择头像，使房间内玩家更容易识别。

---

# 二、目标

让用户在创建房间或加入房间时选择一个卡通头像。

头像应进入房间事件和 replay，确保所有用户看到同一位玩家的头像一致。

目标体验：

```text
首页创建/加入房间
→ 输入昵称
→ 选择头像
→ 进入房间
→ 房间玩家总分条显示头像
→ 高频记录玩家选择区显示头像
```

---

# 三、范围约束

## 不改变

- 不上传头像；
- 不做头像裁剪；
- 不做自定义头像；
- 不新增依赖；
- 不新增 Redis Key；
- 不改变计分规则；
- 不改变同步机制；
- 不改变 Event Sourcing 架构。

## 可以改变

- `CreateRoomRequest`；
- `JoinRoomRequest`；
- `RoomPlayer`；
- `PLAYER_JOINED` payload；
- replay 恢复玩家时携带头像；
- 首页创建/加入 UI；
- 房间页玩家展示组件。

---

# 四、数据设计

新增：

```ts
avatarId?: string
```

位置：

- `RoomPlayer.avatarId`
- `CreateRoomRequest.avatarId`
- `JoinRoomRequest.avatarId`
- `PLAYER_JOINED` event payload

兼容策略：

- `avatarId` 可选；
- 旧房间没有头像时使用前端 fallback；
- replay 读取不到头像时不影响玩家加入；
- 后端校验只允许已知头像 id，非法值 fallback 到默认头像或返回校验错误。

本阶段优先采用：

```text
非法 avatarId 直接使用默认头像
```

避免因为旧客户端或异常输入影响加入房间主流程。

---

# 五、头像资源

头像来源：

```text
frontend/src/assets/avatars/*.svg
```

要求：

- 建立前端 avatar registry；
- 每个头像有稳定英文 id；
- UI 展示中文 label；
- 不直接把文件名散落在页面组件中。

建议组件：

```text
frontend/src/utils/avatars.ts
frontend/src/components/AvatarSelector.tsx
frontend/src/components/room/PlayerAvatar.tsx
```

# 七、验收标准

功能验收：

- 创建房间前可选择头像；
- 加入房间前可选择头像；
- 创建者进入房间后显示所选头像；
- 加入者进入房间后显示所选头像；
- 其他用户看到该玩家头像一致；
- 未选择或旧房间玩家有默认头像；
- 刷新页面后头像不丢失；
- replay 后头像仍存在。

代码验收：

- 不使用 `any`；
- 不新增依赖；
- 旧房间兼容；
- `npm run typecheck` 通过；
- `npm run lint` 通过；
- `npm run build` 通过。

截图验收：

- 375px 首页头像选择；
- 375px 房间玩家总分条；
- 375px 高频记录玩家区；
- 430px 房间玩家总分条；
- 长昵称下头像和昵称不挤压。

# 八、不能接受的结果

以下情况视为任务未完成：

- 头像只存在 localStorage，其他用户看不到；
- 头像没有进入 Event Log；
- 旧房间没有 avatarId 时崩溃；
- 首页头像选择占据过多首屏空间；
- 玩家总分条出现文字或图片挤压；
- 为头像功能新增后端存储模型；
- 修改计分规则或同步机制。

---

# 九、最终原则

头像是玩家识别辅助，不是社交资料系统。

本阶段只做：

- 选择；
- 保存；
- replay；
- 展示。

保持简单、稳定、可维护。
