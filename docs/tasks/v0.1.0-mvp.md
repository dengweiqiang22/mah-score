# 开发任务书（TASK）

> 本文档定义 mah-score 当前版本（MVP）的开发任务。
>
> AI Agent（Codex）应按照本文档的任务顺序完成开发。
>
> 如本文档与 AGENTS.md、ARCHITECTURE.md、DECISIONS.md 存在冲突，以后三者为准。

---

# 一、当前版本

**版本：v0.1.0 MVP**

目标：

完成一个可实际使用的四川血战到底多人计分系统。

要求：

- 可部署到 Vercel
- 支持多人同步
- 支持完整一场麻将计分
- 支持撤销
- 支持事件回放（Replay）

不追求复杂功能。

优先完成 MVP。

---

# 二、开发原则

必须遵循：

✅ 小步提交

✅ 一次完成一个 Epic

✅ 每完成一个 Epic 必须能够运行

✅ 不得一次实现多个 Epic

复杂需求：

必须先分析方案，再开始开发。

# 八、Definition of Done（DoD）

每完成一个 Epic，必须满足：

□ TypeScript 无错误

□ ESLint 无错误

□ 能正常运行

□ 不影响已有功能

□ 更新必要文档（如接口、架构变更）

□ 输出本次修改总结

不得进入下一个 Epic，直到当前 Epic 完成。

---

# 九、禁止事项

修改技术栈

修改 Event Model

修改 Redis 数据结构

增加新的数据库

增加新的基础设施

增加新的框架

不得新增依赖。
