# 开发任务书（TASK）

> 本文档定义 mah-score v0.13.4 的候选开发任务。
>
> v0.13.4 的目标是在房间页高频记录清晰度优化后，继续整理首页入口体验，并补齐分享与历史局明细中的两个可用性问题。
>
> 如本文档与 AGENTS.md、ARCHITECTURE.md、DECISIONS.md 存在冲突，以后三者为准。

---

# 一、当前版本

**版本：v0.13.4**

主题：

首页入口与分享细节优化。

背景：

v0.13.0 - v0.13.3 已完成：

- 房间页产品化；
- 低频功能收纳；
- 玩家头像字段和基础展示；
- 高频记录页清晰度优化；
- 事件图标和事件明细降噪。

但仍有以下问题：

- 首页 / 加入房间页仍偏表单化，首屏产品识别和操作路径还可以继续优化；
- 邀请二维码只能静态展示，手机上不方便点开放大给他人扫码；
- 历史局展开后的撤销图标偏小，且缺少底色，触达成本偏高；
- 新头像资源尚未定稿，本轮不替换头像资源。

---

# 二、目标

本次目标是优化用户进入和分享链路：

- 首页更像成熟的轻量麻将计分工具，而不是普通表单；
- 创建房间、加入房间、邀请链接进入三种路径更清晰；
- 二维码可点击放大，方便线下分享扫码；
- 历史局事件里的撤销入口更容易识别和点击；
- 继续保持 Mobile First。

---

# 三、范围约束

## 不改变

- API；
- Event Sourcing；
- Redis 数据模型；
- domain 计分逻辑；

- 计分规则；

- 玩家头像数据模型；
- 当前头像资源映射。

## 暂缓

头像资源替换暂缓。

用户放入的以下资源暂不接入、不提交：


原因：

头像还没有定稿，等资源确认后再单独做头像资源替换任务。

- 首页创建 / 加入房间表单布局；
- 邀请链接模式下的首页展示；
- 头像选择区视觉层级，但不替换资源；
- 二维码展示方式；
- 二维码放大 Dialog；
- 历史局事件明细的撤销按钮尺寸和样式；
- 相关产品级组件拆分。

# 四、设计原则

## 首页不是后台表单

首页应服务于牌桌旁快速进入房间。

优先展示：

```text
品牌识别
加入 / 创建入口
昵称
房间号
头像选择
主操作按钮
```

要求：

- 不做营销页；
- 不做大面积装饰；
- 不堆多个同重量 Card；
- 主按钮足够明确；
- 邀请链接进入时，优先展示加入房间路径；
- 375px 竖屏下核心操作尽量在首屏内完成。

## 二维码可以放大

当前二维码在“更多 / 邀请加入”中静态展示。

问题：

- 手机上二维码偏小；
- 递给别人扫码时不够方便；
- 点击二维码没有反馈。

要求：

- 二维码区域应表现为可点击；
- 点击后打开 Dialog；
- Dialog 中展示更大的二维码；
- Dialog 保留房间号和邀请链接的必要辅助信息；
- 支持关闭；
- 不新增依赖；
- 保留现有复制和分享按钮。

建议使用：

```text
Dialog
QrCode
Copy
Share2
X
```

## 历史局撤销入口更易点击

当前历史局事件明细中的撤销图标偏小，且没有明显底色。

问题：

- 触控目标偏小；
- 红色图标孤立，识别度不足；
- 在历史局明细中容易被忽略。

要求：

- 撤销按钮使用略大的图标；
- 按钮必须有浅红底色；
- 点击目标至少接近 36px；
- 保持低权重，不能变成整行最抢眼的危险按钮；
- 已撤销状态仍显示轻量 Badge；
- 当前局详情和历史局展开后的事件明细应保持一致。

建议：

```text
h-9 w-9
bg-red-50
text-red-700
active:bg-red-100
Undo2 h-4 w-4
```

```text

```

分享区域可根据现有代码抽取或调整：

```text

```

```text

```

组件命名可根据现有代码微调，但组件应表达产品语义，不要只表达 HTML 结构。


功能验收：

- 首页创建房间流程可用；
- 首页加入房间流程可用；
- 邀请链接进入时加入路径清晰；
- 头像选择仍可用，但不替换资源；
- 二维码点击后可以放大；

- 历史局撤销入口更容易点击；
- 已撤销事件状态展示正常。

视觉验收：

- 首页不再像简单表单堆叠；
- 375px 竖屏下核心进入流程清晰；
- 二维码可点击状态明确；
- 大二维码适合手机面对面扫码；
- 历史局撤销按钮有底色，图标略大；
- 危险操作没有过度抢占页面视觉。

代码验收：

- 不新增依赖；
- 不使用 `any`；
- 不改变 domain 与 API；
- 不提交未定稿头像资源；
- `npm run lint` 通过；
- `npm run test` 通过；
- `npm run typecheck` 通过；
- `npm run build` 通过。

# 八、不能接受的结果

以下情况视为任务未完成：

- 头像资源被提前替换或提交；
- 首页只是换颜色，没有重组信息架构；
- 邀请链接模式仍然不清楚；
- 二维码不能点击放大；
- 二维码 Dialog 太小，不适合扫码；
- 撤销图标仍然是无底色的小图标；
- 历史局与当前局事件明细样式割裂；
- 为 UI 细节新增不必要依赖；
- 修改了计分规则或 Event Sourcing 逻辑。
