# 开发任务书（TASK）

> 本文档记录 mah-score v0.12.1 的补充迭代任务。
>
> v0.12.1 的目标是在 v0.12.0 固定高频录入工作台基础上，合并事件主体玩家区与责任玩家区，改为“单一玩家区 + 状态驱动语义”。
>
> 如本文档与 AGENTS.md、ARCHITECTURE.md、DECISIONS.md 存在冲突，以后三者为准。

---

# 一、当前版本

**版本：v0.12.1**

主题：

合并高频录入玩家角色选择。

目标：

消除“玩家”和“点炮玩家”两套重复玩家网格，让同一个玩家区根据录入状态动态承担主体玩家和责任玩家选择。

核心目标：

- 页面只保留一组玩家卡片。
- 玩家按钮、事件按钮、番数按钮保持固定位置。
- 点炮责任玩家在原玩家区选择。
- 直杠引杠玩家在原玩家区选择。
- 主体玩家与责任玩家用不同视觉状态表达。
- 番数只在需要番数的状态启用。
- 提交成功后清空录入状态。
- 不修改服务端接口、计分算法、同步逻辑和数据模型。

---

# 二、背景说明

v0.12.0 已经完成固定高频录入工作台：

```text
玩家
事件
番数
反馈
```

但页面仍存在角色选择表达不够收敛的问题：

- 主体玩家和责任玩家容易被拆成两套视觉区域。
- 同一批玩家重复出现会增加页面高度。
- 用户需要在上下两个玩家区之间移动手指。
- 玩家位置无法形成稳定肌肉记忆。
- 后续扩展直杠责任玩家时可能继续增加重复区域。

因此 v0.12.1 将玩家选择收敛为：

```text
单一玩家区 + 状态驱动语义
```

---

# 三、本次开发范围

## 必须完成

状态机重构：

- 使用显式状态机表达录入阶段。
- 状态必须覆盖 `idle`、`actor_selected`、`selecting_counterparty`、`selecting_fan`、`liuju_mode`。
- 避免继续依赖 `selectedPlayer`、`selectedDianpaoPlayer`、`selectedEvent`、`selectedFan` 这类松散组合。
- 同一玩家不能同时成为主体玩家和责任玩家。

玩家区复用：

- 删除独立“点炮玩家”选择区域。
- 不新增“放杠玩家”独立区域。
- 点炮、直杠都复用同一组玩家卡片选择责任玩家。
- 玩家顺序在状态变化时不改变。
- 状态变化不导致页面整体跳动。

玩家卡片角色样式：

- 同一个玩家卡片组件支持 `default`。
- 同一个玩家卡片组件支持 `actor`。
- 同一个玩家卡片组件支持 `counterparty`。
- 同一个玩家卡片组件支持 `disabled`。
- 角色不能只靠颜色区分，必须同时使用角色文字、边框或背景。

番数区逻辑：

- 番数区固定占位。
- 仅 `selecting_fan` 状态允许选择番数。
- 自摸流程为：玩家 -> 自摸 -> 番数。
- 点炮流程为：胡牌者 -> 点炮 -> 点炮者 -> 番数。
- 直杠、补杠、暗杠不携带无意义番数。

取消与重置：

- 提供明确的取消当前录入机制。
- 再次点击主体玩家可以取消选择。
- 提交成功后恢复 `idle`。
- 提交成功后显示轻量成功反馈。
- 提交成功后提供撤销上一条入口。

## 不开发

新的计分规则。

新的番数体系。

新的后端接口。

新的 Event 类型。

新的 Redis Key。

Bottom Sheet 责任玩家选择。

新的全局状态管理库。

# 四、目标交互

## 初始状态

```text
提示：请选择玩家
玩家区：所有未胡牌玩家可点击
事件区：固定展示
番数区：固定展示但禁用
```

## 已选主体玩家

```text
提示：已选择 raisin，请选择事件
玩家区：raisin 显示主体玩家样式
事件区：可点击
番数区：禁用
```

## 点炮

```text
选择 raisin
-> 点击点炮
-> 原玩家区选择张三
-> 选择番数
-> 自动提交
```

要求：

- raisin 保持主体玩家样式。
- raisin 禁用，不能作为点炮者。
- 张三显示责任玩家样式。
- 不出现第二组玩家列表。

## 直杠

```text
选择 raisin
-> 点击直杠
-> 原玩家区选择张三
-> 自动提交
```

要求：

- 保持当前业务规则，直杠继续记录引杠玩家。
- raisin 保持主体玩家样式。
- 张三显示责任玩家样式。
- 不新增“放杠玩家”区域。

## 自摸

```text
选择 raisin
-> 点击自摸
-> 选择番数
-> 自动提交
```

要求：

- 番数只在点击自摸后启用。
- 玩家区锁定当前录入，不允许误切换。

## 流局

```text
点击流局
-> 进入 liuju_mode
-> 确认流局
```

要求：

- 流局仍保留确认。
- 不引入查花猪、查大叫、退刮风下雨。

# 八、后续注意事项

后续新增其他需要责任玩家的事件时，必须继续复用同一个玩家网格。

禁止为了表达新角色再新增第二套玩家卡片区。

如需扩展角色样式，应优先扩展 `PlayerVisualState` 或角色标签，而不是复制组件。
