# 开发任务书（TASK）

> 本文档定义 mah-score v0.14.0 的候选开发任务。
>
> v0.14.0 的目标是在 v0.13.4 完成首页、分享和房间页局部信息降噪后，继续优化房间页高频记录区的操作清晰度。
>
> 如本文档与 AGENTS.md、ARCHITECTURE.md、DECISIONS.md 存在冲突，以后三者为准。

---

# 一、当前版本

**版本：v0.14.0**

主题：

高频记录区交互清晰度优化。

背景：

v0.13.4 已完成：

- 事件明细撤销按钮增强；
- 本局结算顶部重复摘要移除；
- 顶部玩家分数收纳到头像底部；
- 头像展示边框和裁切优化。

从当前高频记录区截图观察，仍有以下问题：

- “当前步骤”提示框占用高度偏大；
- 玩家卡片看起来偏静态，不够像可点击按钮；
- 玩家卡片中的分数与顶部总分条重复；
- 六个事件按钮权重完全一致，高频操作不够突出。

---

# 二、目标

本次目标是让高频记录区更像牌桌旁的快速操作台：

- 当前步骤提示改成一行紧凑提示；
- 玩家选择入口更明确；
- 减少玩家卡片中的重复分数信息；
- 高频事件入口更容易被一眼识别；
- 继续保持 Mobile First。

---

# 三、范围约束

## 不改变

- API；
- Event Sourcing；
- Redis 数据模型；
- domain 计分逻辑；
- 录入状态机；
- 计分规则；
- 同步机制；
- 玩家头像数据模型；
- 当前头像资源映射。

## 可以改变

- 高频记录区当前步骤提示样式；
- 玩家卡片视觉状态；
- 玩家卡片内部信息层级；
- 事件按钮视觉权重；
- 高频记录区局部组件结构。

---

# 四、实施顺序

## 第一轮：当前步骤提示紧凑化

目标：

将当前绿色提示框改为一行紧凑提示，减少首屏高度占用。

要求：

- 保留现有状态文案来源；
- 不新增业务状态；
- 不改变录入状态机；
- 文案只表达下一步动作；
- 375px 竖屏下不显得拥挤。

建议：

```text
请选择玩家
请选择事件
请选择点炮玩家
请选择番数
确认后记录本次事件
```

可以考虑展示为：

```text
当前：请选择玩家
```

或直接合并到区域标题右侧：

```text
玩家 · 请选择玩家
```

具体实现以现有页面结构为准。

## 第二轮：玩家卡片可点击感

目标：

让玩家卡片更明确地表现为可点击选择入口。

要求：

- 未选中状态保持轻量；
- 选中状态使用浅绿底和绿色边框；
- active 状态有轻微反馈；
- 不增加复杂动画；
- 不影响已胡牌、当前玩家等状态表达。

## 第三轮：玩家卡片信息降噪

目标：

减少玩家卡片与顶部总分条的信息重复。

要求：

- 优先保留头像和昵称；
- “我”标签保留；
- 分数如继续展示，应降低视觉权重；
- 如移除分数，必须确认不会影响选择玩家时的判断。

## 第四轮：事件按钮权重优化

目标：

让高频事件入口更容易被快速识别。

要求：

- 自摸、点炮可略微增强；
- 流局保持低权重；
- 六个按钮仍保持统一栅格和触控尺寸；
- 不改变事件类型和提交逻辑。

---

# 五、验收标准

功能验收：

- 玩家选择流程可用；
- 事件选择流程可用；
- 点炮、自摸、杠、流局流程不受影响；
- 已胡牌玩家状态仍能正确表达；
- 录入状态提示准确。

视觉验收：

- 当前步骤提示不再占用大块高度；
- 375px 竖屏下玩家区和事件区更紧凑；
- 玩家卡片明显可点击；
- 选中状态清晰；
- 事件按钮仍易扫视和单手点击。

代码验收：

- 不新增依赖；
- 不使用 `any`；
- 不改变 domain 与 API；
- `npm run lint` 通过；
- `npm run typecheck` 通过；
- `npm run build` 通过。

---

# 六、不能接受的结果

以下情况视为任务未完成：

- 为 UI 优化修改计分规则；
- 为 UI 优化修改 Event Sourcing 逻辑；
- 当前步骤提示被完全删除，导致用户不知道下一步；
- 玩家选中态不清楚；
- 事件按钮触控面积变小；
- 页面视觉变得更拥挤；
- 新增不必要依赖。
