# 四川麻将计分规则（v0.2.0）

> 本文档从 `docs/四川麻将（血战到底）规则文档.md` 中提取 mah-score 需要实现的计分规则。
>
> mah-score 是计分工具，不是麻将游戏引擎。因此本文档只描述录入、计分、Replay 和结算需要关心的规则。

---

# 一、规则定位

v0.2.0 只实现真实牌局中的人工计分。

系统不判断玩家是否真的可以胡牌，也不判断手牌是否合法。

以下内容由线下玩家自行确认：

- 是否缺门
- 是否听牌
- 是否满足胡牌牌型
- 是否允许抢杠
- 是否过水不能胡
- 是否进入最后四张自动胡

系统只负责：

- 记录计分事件
- 校验事件中的玩家和基础字段是否合法
- 根据事件 Replay 得到分数
- 支持 Undo
- 支持历史复盘和结算

---

# 二、基础概念

## 基础分

v0.2.0 使用 1 分作为基础分。

## 番数

胡牌事件支持番数：

```text
1 番、2 番、3 番、4 番
```

4 番视为封顶。

计分倍数：

```text
1 番 = 1 分
2 番 = 2 分
3 番 = 4 分
4 番 = 8 分
```

计算方式：

```text
score = 2 ^ (fan - 1)
```

其中：

```text
fan 最小为 1
fan 最大为 4
```

旧版本事件如果没有 `fan` 字段，按 1 番处理。

# 三、胡牌计分

## 点炮胡

点炮胡表示：

```text
其他玩家打出的牌使自己胡牌
```

计分规则：

```text
放炮玩家支付给胡牌玩家 fan 对应分数
```

示例：

```text
A 点炮给 B，B 胡 3 番

A: -4
B: +4
```

## 自摸

自摸表示：

```text
自己摸到胡牌所需的牌
```

计分规则：

```text
其他未胡玩家分别支付给自摸玩家 fan 对应分数
```

示例：

```text
B 自摸 2 番，A、C、D 均未胡

A: -2
C: -2
D: -2
B: +6
```

## 杠上开花

杠上开花在 v0.2.0 中按自摸处理。

系统不单独识别牌局过程，只允许在自摸事件中记录备注或扩展标记。

## 一炮多响

一炮多响表示：

```text
同一张牌使多个玩家胡牌
```

v0.2.0 允许多次记录点炮胡事件，且这些事件可以拥有同一个放炮玩家。

计分规则：

```text
放炮玩家分别支付给每个胡牌玩家对应分数
```

示例：

```text
A 点炮，B 胡 2 番，C 胡 3 番

A: -6
B: +2
C: +4
```

# 四、血战到底

血战到底的核心规则：

```text
一家胡牌后，本局不立即结束
```

本局继续进行，直到：

- 3 家已经胡牌
- 或牌墙摸完流局

v0.2.0 的 Replay 规则：

- 每一局允许最多 3 名不同玩家胡牌
- 已经胡过的玩家不能在同一局再次胡牌
- 第 3 名玩家胡牌后，本局结束并进入下一局
- 流局事件会结束当前局并进入下一局

---

# 五、杠牌计分

## 杠牌类型

v0.2.0 支持三类杠牌计分：

- 直杠
- 补杠
- 暗杠

其中：

```text
直杠、补杠属于刮风
暗杠属于下雨
```

## 直杠

直杠表示：

```text
手中已有三张相同牌，其他玩家打出第四张
```

计分规则：

```text
引杠玩家支付给杠牌玩家 1 分
```

示例：

```text
A 打出牌，B 直杠

A: -1
B: +1
```

## 补杠

补杠表示：

```text
之前已经碰牌，后来自己摸到第四张
```

计分规则：

```text
其他未胡玩家分别支付给杠牌玩家 1 分
```

示例：

```text
B 补杠，A、C、D 均未胡

A: -1
C: -1
D: -1
B: +3
```

## 暗杠

暗杠表示：

```text
自己手中形成四张相同牌
```

计分规则：

```text
其他未胡玩家分别支付给杠牌玩家 2 分
```

示例：

```text
B 暗杠，A、C、D 均未胡

A: -2
C: -2
D: -2
B: +6
```

# 六、流局

v0.2.0 支持流局事件。

流局事件的作用：

```text
结束当前局并进入下一局
```

v0.2.0 暂不实现以下流局结算：

- 查花猪
- 查大叫
- 退刮风下雨

原因：

这些规则依赖手牌、听牌状态和最大可胡番型。当前项目是计分工具，不记录完整牌局过程，暂不适合在 v0.2.0 中实现。

---

# 七、Undo

Undo 必须遵守 Event Sourcing 原则。

规则：

- 不删除历史 Event
- 不修改历史 Event
- 追加一个 Undo Event
- Replay 时识别 Undo Event
- 被撤销的目标 Event 不参与分数和局数计算
- 被撤销的目标 Event 在历史记录中仍然可见

---

# 八、事件范围

v0.2.0 需要支持的计分事件：

- 点炮胡
- 自摸
- 直杠
- 补杠
- 暗杠
- 流局
- 撤销
- 结束游戏

v0.2.0 不实现的事件：

- 发牌
- 摸牌
- 出牌
- 碰牌
- 吃牌
- 抢杠判定
- 过水判定
- 查花猪
- 查大叫
- 退税
- 自动算番

---

# 九、API 和 Replay 要求

API 负责基础合法性校验：

- 房间必须存在
- 房间必须处于 PLAYING
- 玩家必须存在
- 点炮胡的放炮玩家不能等于胡牌玩家
- 直杠的引杠玩家不能等于杠牌玩家
- 番数必须在 1 到 4 之间
- 同一玩家不能在同一局重复胡牌

Replay 负责业务状态计算：

- 当前局数
- 当前局已胡玩家
- 每个玩家总分
- 胡牌次数
- 点炮次数
- 杠牌次数
- 每条事件的分数变化
- Undo 后的有效状态

所有业务状态必须来自 Event Replay。
